You own a license plate manufacturing company. Write a program that takes a population and determines the simplest pattern that will produce enough unique plates. Since all the plates that match the pattern will be generated, find the pattern that produces the least excess plates. Use a combination of letters (A-Z) and numbers (0-9).

Enter a population size!

Simply enter a population size (a number) in the field below and hit <enter> and the result will be displayed in the list to the right.

My Solution

To solve the license plate problem, I first implemented a loop that would solve the problem using only numbers (no letters). From this I could take the count of numbers (N_POWER) required to make sufficient license plates. The total number plates created by this process was also saved into a variable as a reference as the "best result".

I then looped N_POWER times, starting at 10^0 each time adding a power of 10 until 10^N_POWER. Inside the iteration I would make the remaining plates (if any) by incrementally adding powers of 26 (Letters). The internal loop breaks when the total number of plates created exceeds or is equal to that of the population and the new value is checked against the "best result" to determine if a new best pattern has been found.
